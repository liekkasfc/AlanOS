# Alan OS RFCs

Read the RFCs in numeric order. Each document builds one layer of the personal revenue operating system.

1. [`RFC-0000: Foundation`](RFC-0000-FOUNDATION.md) - why Alan OS exists and what it must not become.
2. [`RFC-0001: Vision`](RFC-0001-VISION.md) - long-term direction and product principles.
3. [`RFC-0002: Domain Model`](RFC-0002-DOMAIN_MODEL.md) - core entities from Signal to AlanMemory.
4. [`RFC-0003: Workflow`](RFC-0003-WORKFLOW.md) - the operating loop from discovery to learning.
5. [`RFC-0004: Schema`](RFC-0004-SCHEMA.md) - schema contract map, with JSON Schemas in [`schemas/`](../../schemas).
6. [`RFC-0005: Daily Output`](RFC-0005-DAILY_OUTPUT.md) - the daily action packet centered on Today's Bet.
7. [`RFC-0006: Data Sources`](RFC-0006-DATA_SOURCES.md) - source rules for finding gaps without becoming a news aggregator.
8. [`RFC-0007: Alan Context`](RFC-0007-ALAN_CONTEXT.md) - personal context used to score fit and choose executable bets.

During Sprint 0.1, all RFCs are Draft. Treat them as design constraints for manual validation before writing collectors or business logic.
